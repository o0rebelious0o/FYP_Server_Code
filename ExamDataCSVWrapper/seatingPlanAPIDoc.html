Seating Plan Data API

What Is This?

This API has been written to provide an easy to use JSON data source for the exam Seating Plans.

The API is available at http://146.169.44.162:8080/seatingPlan or https://146.169.44.162:8443/seatingPlan for the HTTPS version.

Returned Data Format

This API returns JSON data for ease of use.  The returned data contains the following standard information:

  • dateProduced - the date the original csv data was produced
  • sessionStart - the beginning of the period of time seating information is within
  • sessionEnd- the end of the period of time seating information is within
  • {seating_plans} - this element contains the representation of the exam data according to the view requested.  The value of data keys will be explained in the section below.  To explain, x : y is data with x as the key, y as the data.  [] represents an array of JSON objects, and {x : y, a : b}  represents a JSON object containing two keys, x and a, which have values y and b respectively.  $key means that there will be an arbitrary number of keys contained in the object, populated by the data itself;  these can be iterated over in the receiveing code.  key represents the nature of the data making up the key.  This data is returned grouped by date, then room.

All exams are represented with the headers from the original CSV data:

  • 				Date
  • 				Time
  • 				Room(s)
  • 				Course
  • 				Class
  • 				Seat
  • 				CID

API Parameters

SessionStart/SessionEnd

?sessionStart=2014-03-18T00%3A00%3A00.000Z&sessionEnd=2014-04-29T15%3A30%3A00.000Z

SessionStart/SessionEnd is a JSON (ISO8601) formatted dateTime string.  These parameters allow the specification of a session windows which only seating information within this window will be returned.

These variables default to the earliest and latest representable times possible if left unspecified, encapsulating any seating information possible in the original data source.

Room

?room=344

The room parameter allows specification of a room causing only seating information for those taking their exam in this room to be returned.

Course

?course=c350

The course parameter allows specification of a course causing only seating information for those sitting this module to be returned.

Other Information

These API parameters can be combined in any conbination to filter and select specific seating information from the original source.  If no parameters are specified, all will assume default values to show all original data from the source.
